# PRガイド
本記事では、Pull Request(以降、PR)の作成方法について説明します。

## PRの作成
以下の場合に使用してください。
- [1] 自分のforkしたレポジトリからdevにマージする時や、
- [2] devからmainにマージする時

## PRのタイトル
```
<category> <description>
```

### `<category>`
- `Fix:`
何かしらのバグなどの修正。

- `Feat:`
新機能の追加。

- `Docs:`
ドキュメントの修正/追加。

### `<description>`
**日本語**で説明を記述してください。

## PRのタグ
Pull Requestを実行する際に、タグが関連すると思う場合、タグを付与してください。

---
## PRのマージ
PRのマージには、基本的には自分を除く
- [1] 1人以上
- [2]　2人以上
のレビューを必要とします。

レビューが必要な場合には、Discordサーバーの(開発部　-> レビュー依頼（WEB)）にPRのリンクを貼ってください。

また、PRのマージの際にはデフォルトの英語のメッセージのままで構いません
[参照](/commit.md#例外)
